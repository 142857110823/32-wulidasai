## 项目定位

FreshSalt Surface 是面向大学物理竞赛的 Android 优先 Flutter APP 原型，核心不是“拍照识别工具”也不是“通用分析平台”，而是：

- 受控 RGB 成像
- ROI 与灰卡质控
- 半物理 / 灰箱模型
- 模拟数据验证
- 后续真实实验接入

## 三大核心问题

1. 手机拍照能力是否真实存在
2. 图像处理是否真实基于像素而非 metadata
3. UI 是否在浏览器里呈现出完整平台观感

---

## 方案一：手机拍照能力

### 当前判断

当前只能确认存在“模拟拍照 / mock I0 I1”链路，不能确认真实手机拍照、权限申请、相机预览、文件落盘这条闭环真的成立。

### 任务

- 确认真实相机依赖、权限声明、预览入口、拍照结果路径
- 确认是否真的能从设备相机拿到 I0 / I1
- 识别当前是否只是按钮占位或模拟图片

### 要求

- 必须有真实相机依赖与权限链路
- 必须有浏览器或真机可见证据
- 不能只证明“按钮存在”

### 调用 skills

- `using-superpowers`
- `freshsalt-platform-priority-guard`
- `verification-before-completion`

### 验证标准

- 依赖存在
- 权限链路存在
- 真机或浏览器可见预览存在
- I0 / I1 真实输入存在

### 验收条件

- 能证明“可以拍”，不是“有个入口”

---

## 方案二：真实图像处理能力

### 当前判断

当前证据指向 metadata 驱动、mock diff 路径、模拟特征值，不足以证明真实像素级处理。

### 任务

- 确认图像是否被解码
- 确认 ROI 是否来自真实像素坐标裁剪
- 确认灰卡、质控、特征提取是否直接来自图像内容
- 确认结果是否来自真实输入链路而不是字典映射

### 要求

- 必须覆盖输入图像 -> ROI -> 质控 -> 特征 -> 结果
- 必须区分 `simulated` 和 `real`
- 不能把 metadata 当像素

### 调用 skills

- `using-superpowers`
- `verification-before-completion`

### 验证标准

- 有图像解码证据
- 有 ROI 裁剪证据
- 有像素级质控证据
- 有真实特征提取证据
- 有结果与图像内容的可追溯证据

### 验收条件

- 能证明“处理的是图”，不是“处理描述字段”

---

## 方案三：UI 平台观感

### 当前判断

当前必须以浏览器真实可见结果为准，不接受“路由存在”“页面存在”“组件存在”当作完成。

### 任务

- 首页第一屏先呈现完整平台感
- 主链路页面要像实验工作台，不像说明书
- 结果页、质控页、首页的层级与基线对齐

### 要求

- 先看浏览器可见差异
- 每次只修一个最高优先级可见偏差
- 不先补测试，不先扩模块

### 调用 skills

- `using-superpowers`
- `freshsalt-platform-priority-guard`
- `verification-before-completion`

### 验证标准

- 浏览器里能看到完整首页骨架
- 主链路页面可见且结构清晰
- 结果页不是空白页
- 模拟数据持续有“模拟数据”标记

### 验收条件

- 只要浏览器看起来不像完整平台，就继续修 UI
