# 真实图片处理核心问题方案

## 当前进度

- 已确认当前质控、特征、差分与预测链仍以 mock / metadata 驱动为主。
- 当前结果更适合作为平台演示，不足以证明真实像素链已打通。
- 模拟链路已具备后续替换为真实图像处理的结构基础。

## 待完成工作

- 把当前 mock 驱动链替换为真实像素处理链。
- 让 ROI、灰卡质控、特征提取、差分图生成都来自真实图像输入。
- 在不破坏现有平台 UI 的前提下完成底层升级。

## 执行步骤

1. 明确真实输入对象。
   先限定处理对象为真实 I0、I1 图像与真实 ROI 数据，不在第一轮扩到更多样本域。

2. 打通真实图像读取。
   让服务层消费真实图像数据，而不是仅消费 demo metadata。

3. 替换质控来源。
   曝光、清晰度、灰卡稳定性、ROI 合法性必须改由真实图像计算得出。

4. 替换特征提取来源。
   色彩差分、白化、高光、纹理等关键特征应由真实像素处理结果生成。

5. 保留模拟链作为并行基线。
   在真实链未完全稳定前，保留 mock 模式用于平台演示与回归对照。

## 验证标准

- 系统可以读取真实 I0 / I1 图像并进入处理链。
- ROI、灰卡、质控指标来自真实图像，不再只是 metadata 注入值。
- 特征向量与差分结果来自真实图像计算。
- 模拟数据模式仍明确标识，真实模式与模拟模式不混淆。
- 未运行真实链时，明确写未运行，不伪造结果。

## 验收条件

- 真实图像输入已进入质控、特征、预测链。
- 结果页展示的数据来源可以被追溯到真实像素处理过程。
- 模拟链与真实链边界清晰，不相互伪装。
