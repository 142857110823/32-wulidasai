# FreshSalt Surface App Cognition

## 一句话判断

FreshSalt Surface 已经不是空壳原型，而是具备平台骨架、主链路和模拟闭环的实验型 Flutter APP；当前最需要收紧的是浏览器可见的第一屏平台感、文案稳定性和主链统一性。

## 已确认事实

### 1. 项目边界明确

- 这是大学物理竞赛场景下的实验型 APP。
- 不做食品安全判定，不做执法检测，不混入 `water_erosion_mvp`。
- 当前阶段以模拟数据、模拟图像、模拟模型包验证整条闭环。

### 2. 工程骨架已存在

入口链路已经明确：

- `main.dart`
- `app.dart`
- `routing/app_router.dart`

主模块也已覆盖：

- 首页 `HomePage`
- 采集/质控/ROI/预测/结果链路
- 历史页 `HistoryPage`
- 分析页、报告页、设置页
- 点击验证台 `DemoValidationPage`

### 3. 数据和服务层不是空的

`AppScope` 已经装配了：

- 模型包服务
- 质控服务
- 特征提取服务
- 预测服务
- 点击验证服务
- 会话仓储
- 导出服务
- orchestrator 编排器

这说明当前问题不是“没有架构”，而是“架构还没在浏览器里形成稳定平台观感”。

### 4. 模拟闭环语义完整

基准和代码都表明当前演示链路已有：

- 模拟模型包
- 模拟 I0 / I1
- ROI
- 低 / 中 / 高 / 过曝演示案例
- 预测范围、警告、置信度、模拟标记

这让后续工作可以围绕“平台如何讲清楚”推进，而不是先补真实实验数据。

## 当前理解

### 1. 核心价值不是“算出一个数”

这个 app 的价值不只是输出 `mg/cm2 NaCl eq.`，而是把整套实验平台讲明白：

- 当前模式是什么
- 当前样品和采集处于哪一步
- 结果如何得到
- 风险边界在哪里
- 为什么这只是实验验证

### 2. 首页应是平台入口，不是目录页

首页需要同时承担：

- 项目边界和当前模式说明
- 主链入口
- 次级模块分发

它的职责是让用户第一眼知道现在该做什么、下一步去哪。

### 3. 采集主链是可信度核心

主链已经很清楚：

- 质控
- I0
- I1
- ROI
- 特征
- 预测
- 保存

后续判断都应围绕这条链展开。只要这条链在浏览器里不够清楚，其他模块再多也只是次要收益。

### 4. 结果页是第二核心

`ResultPage` 的方向是对的：

- 主结果
- 范围标尺
- 历史趋势
- 模型解释
- 图像证据
- 风险说明

它需要继续朝“实验平台结果详情”靠拢，而不是普通信息拼盘。

## 当前问题

### 1. 高可见文案存在乱码风险

多个 Dart 文件曾出现源码级乱码文本，这会影响：

- 浏览器可读性
- 状态标签准确性
- 页面语义判断

### 2. 浏览器截图和当前源码状态有差异

现有截图仍能看到旧的工作模式入口，而当前默认首页已切到 `HomePage`。
这意味着截图、运行入口和当前代码还没完全对齐。

### 3. 现在更像“很多页”，还不完全像“一个平台”

模块数量和链路逻辑已经不少，但平台感是否成立，仍取决于首页、采集链和结果页是否使用同一种平台语言。

## 当前优点

### 1. 边界健康

项目没有明显跑偏到食品安全或无关平台方向，模拟数据、实验边界、模型范围是一致的。

### 2. 分层合理

页面层、服务层、仓储层、编排层基本已经分开，后续收紧 UI 时不需要重拆逻辑。

### 3. 演示闭环已成型

不是只有界面，而是真的有：

- 质控规则
- 特征向量
- 模型系数
- 结果对象
- 历史存储
- 点击验证矩阵

## 当前最大风险

- UI 与真实运行状态继续脱节。
- 文案编码污染放大后续视觉成本。
- 次级模块抢走首页和主链的注意力。

## 后续思路

### 当前阶段

把它理解成“已有主链逻辑的前端平台校准任务”。

重点不是补更多新模块，而是：

- 确认浏览器真实入口
- 统一首页与主链叙事
- 清理高可见文案与状态层问题

### 执行顺序

1. 首页 / 平台第一屏
2. 质控 -> I0 -> I1 -> ROI -> 特征 -> 预测 -> 保存
3. 结果详情
4. 历史 / 分析 / 报告 / 验证台

### 验收方式

优先基于：

- 浏览器可见结果
- 基准文档
- 当前代码
- 已有截图

不会把“测试通过”直接等同于平台 UI 已完成。

## 总体观点

FreshSalt Surface 已经有值得继续收紧的底子。当前最缺的不是功能总量，而是：

- 浏览器状态和源码状态完全对齐
- 主链叙事和首页平台叙事完全统一
- 文案、状态、模拟标记在高可见页面上稳定一致

只要这三点继续收紧，它就会更像一个完整实验平台，而不是一个功能很多但观感未锁定的原型。
