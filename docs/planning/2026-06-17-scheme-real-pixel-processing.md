# 真实图片处理能力方案

## 当前进度

- 已完成核心服务审查。
- 已确认质控与特征提取当前主要来自 `imageMetadata`，不是来自真实像素。
- 已确认 demo 工厂仍在向主链注入模拟元数据与 mock diff 路径。

## 待完成工作

- 明确真实图像是否被解码。
- 明确 ROI、质控、特征、结果是否真正来自像素内容。
- 用文件级证据和运行级证据同时证明处理能力。

## 执行步骤

1. 锁定最小真实处理闭环。
   必须至少覆盖“输入图片 -> ROI -> 像素级质控 -> 像素级特征 -> 结果输出”。
2. 审查图像输入源。
   检查是否有真实文件读取、图像解码、像素矩阵处理，而不是只传 metadata。
3. 审查 ROI 与质控。
   检查 ROI 是否从图像坐标裁剪；曝光、清晰度、灰卡稳定性是否由真实像素计算。
4. 审查特征提取。
   检查颜色差分、纹理、高光、白化指数是否由真实图像或裁剪图计算。
5. 审查结果输出。
   检查预测输入是否来自真实特征向量，而不是 mock 字段映射。
6. 做运行证据核验。
   若没有真实图片输入样本和可见处理结果，就不能判定满足。

## 验证标准

- 存在真实图像文件读取与解码。
- ROI 裁剪作用于真实图像区域。
- 质控指标来自像素而不是 metadata。
- 特征向量来自真实图像计算。
- 结果页中的图像证据、差分图或摘要可追溯到真实图片内容。

## 验收条件

- 只有当“真实解码 + 真实 ROI + 真实质控 + 真实特征 + 真实结果输入”全部成立，才可判定真实图片处理能力满足。
- 只要仍以 metadata 驱动、mock diff 路径驱动、或无法提供真实输入样本与运行证据，就必须判定未满足。
