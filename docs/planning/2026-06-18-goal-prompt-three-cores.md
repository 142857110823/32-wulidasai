# 适配 Goal 的执行提示词

```markdown
你当前接手的是 `FreshSalt Surface / 表面盐影像助手`，工作目录限定为 `F:\1.大学物理竞赛`，默认中文。

## 一、项目定位

这不是食品安全判断工具，不是执法检测工具，不是“手机随手拍直接测盐”的宣传型 APP。

这是一个面向大学物理竞赛的 Android 优先 Flutter 原型平台，核心目标只有三层：

1. 用平台方式承载受控 RGB 成像实验主链。
2. 用模拟模式先验证交互、流程、模型包、结果页与点击验证闭环。
3. 后续逐步接入真实手机拍照与真实图像处理，而不是推翻 UI 重做。

## 二、本轮只解决三大核心

禁止把注意力放在零碎补丁、次要模块扩张、无关测试堆叠上。

本轮只围绕以下三大核心推进：

1. `UI 是否符合平台定位与基准图`
2. `手机拍照能力是否真正接入`
3. `真实图片处理能力是否真正接入`

## 三、任务拆分

### 任务 A：UI 平台对齐

要求：

- 先看基准文档与浏览器真实页面。
- 不把页面存在、模块存在、路由存在当作 UI 已完成。
- 只修一个最高优先级可见偏差。
- 优先保证首页、采集主链、结果页形成统一平台观感。

必须调用的 skill：

- `using-superpowers`
- `freshsalt-platform-priority-guard`

如涉及设计整理或方案收敛，可额外参考：

- `brainstorming`

验证标准：

- 浏览器中首页第一屏具备完整平台观感。
- 采集主链具备实验测量语义。
- 结果页稳定展示主结果、范围、趋势、解释、边界提示。

### 任务 B：手机拍照能力接入

要求：

- 仅做 Android 优先最小闭环。
- 必须包含权限、真实预览、真实拍摄或真实采集入口。
- 不得破坏现有模拟模式。
- 必须嵌入既有主链，不另起一套流程。

必须调用的 skill：

- `using-superpowers`
- `freshsalt-platform-priority-guard`

如进入真实修复阶段并出现 bug，再调用：

- `systematic-debugging`

验证标准：

- 真实相机权限可请求。
- 真实预览或真实拍摄链可见。
- I0 / I1 的真实采集入口成立。
- 权限失败时有明确阻断反馈。

### 任务 C：真实图片处理能力接入

要求：

- 真实 I0 / I1 图像必须进入处理链。
- ROI、灰卡、质控、特征提取不能再只靠 mock metadata。
- 模拟模式必须继续保留并明确标记。
- 未运行真实链时不得声称已完成。

必须调用的 skill：

- `using-superpowers`
- `freshsalt-platform-priority-guard`

如进入真实修复阶段并出现 bug，再调用：

- `systematic-debugging`

验证标准：

- 真实图像被读取并进入服务链。
- 质控指标来自真实图像计算。
- 特征向量来自真实像素处理。
- 结果来源可追溯，不与模拟数据混淆。

## 四、执行纪律

- 最高优先级永远是：用户原话 > 用户规则/方案/AGENTS/基准文档 > skill > 你的习惯。
- 每次开工先检查当前目录、已有文件、规则文件、可运行命令。
- 必须先读取并遵守：
  - `AGENTS.md`
  - `rules/freshsalt.rules.md`
  - `docs/source/baseline_2026-06-14/大学物理竞赛-app.txt`
  - `docs/source/baseline_2026-06-14/FreshSalt_Surface_APP框架决策树_输入交互模型输出_2026-06-13.md`
  - `docs/source/baseline_2026-06-14/FreshSalt_Surface_APP平台方案与点击式模拟验证规划_2026-06-12.docx`
- 任何小瑕疵、偏差、阻塞，先写入 `outputs/workflow-error-2026-06-15.md`，再更新 `workflow.log`。
- `workflow.log` 只用英文记录，并保持最近 7 天且不超过 30 条。
- 首页和主平台骨架未完全对齐前，禁止优先补测试，禁止拿 `flutter analyze` 或 `flutter test` 当 UI 验收证据。

## 五、最终验收

只有同时满足以下条件，才允许说“三大核心完成”：

1. 浏览器真实可见的首页、采集主链、结果页形成统一平台观感。
2. Android 侧真实手机拍照能力已进入主链。
3. 真实图片处理能力已进入 ROI / 质控 / 特征 / 预测链。
4. 模拟模式持续保留且清晰标记。
5. 所有结论都有文件、代码、命令输出、浏览器结果或设备结果支撑。
```
