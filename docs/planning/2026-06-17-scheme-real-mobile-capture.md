# 手机拍照能力方案

## 当前进度

- 已完成代码级与依赖级审查。
- 已确认当前 APP 仍未具备真实手机拍照能力。
- 已确认当前主链以模拟 I0 / I1、模拟图片路径、模拟元数据为主。

## 待完成工作

- 明确真实相机接入点。
- 明确权限提权链路是否存在。
- 用浏览器或真机可见证据证明“可拍照”而不是“按钮存在”。

## 执行步骤

1. 先锁定输入能力边界。
   当前项目定位是大学物理竞赛实验型 APP，不是随手拍宣传工具；真实拍照必须服务于受控成像、灰卡、ROI 和质控主链。
2. 审查依赖与权限。
   重点核对 `camera`、`image_picker`、`permission_handler`、Android `AndroidManifest.xml`、iOS 权限文案是否存在。
3. 审查路由与服务接线。
   必须看到“相机预览 -> 拍照 -> 生成 I0/I1 文件 -> 进入 ROI / 质控”的真实链路，而不是模拟图片替代。
4. 做可见验收。
   必须以真机、模拟器或浏览器可见页面证明存在真实拍照入口、权限弹窗或真实预览。
5. 若缺任一关键点，直接判定未满足。

## 验证标准

- 存在真实相机依赖和权限声明。
- 存在真实权限请求链路，而不是纯文案占位。
- 存在真实预览或真实拍照动作。
- I0 / I1 来源于真实图像文件，而不是 mock path。
- 浏览器或设备上能看到真实拍照流程证据。

## 验收条件

- 只有当“依赖 + 权限 + 真实输入 + 可见证据”四项同时成立，才可判定手机拍照能力满足。
- 只要仍是模拟按钮、静态图占位、无权限提权路径、或无可见证据，就必须判定未满足。
