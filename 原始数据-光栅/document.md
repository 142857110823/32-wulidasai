# 光栅原始图像数据说明

更新时间：2026-06-25

## 1. 数据定位

本目录保存 FreshSalt Surface / 表面盐影像助手 项目的光栅反射实验原始图片。

当前数据用于判断：

- 光栅反射图像是否存在可重复的光谱/伪光谱响应；
- 不同盐水浸泡浓度是否能在图像特征中形成稳定差异；
- 是否具备进入 PLSR 或其他回归/分类模型的最低数据基础。

当前数据不得直接解释为：

- 水果表面真实盐分含量；
- 食品安全、商品分级或执法检测结论；
- 已经验证成功的盐分无损检测结果。

## 2. 实验流程

实验链路：

```text
光源
-> 圣女果样品
-> 反射光
-> 光栅
-> 拍照记录
```

已知固定条件：

- 样品：同一个圣女果体系下的实验样品；
- 光源到样品/仪器条件：按现场固定光路执行；
- 光学仪器位置 1：320 mm；
- 光栅相关固定位置：580 mm；
- 拍摄对象：透过同一个光栅后的反射光图像；
- 每组包含原生图与聚焦图。

## 3. 分组与标签

共有五个盐水浸泡组：

| 组别 | 标签值 | 标签含义 |
|---|---:|---|
| 对照组 | 0 | 清水浸泡 |
| 低浓度组 | 2.0 | 2.0 mg/L 盐水浸泡 |
| 中低浓度组 | 4.0 | 4.0 mg/L 盐水浸泡 |
| 中高浓度组 | 6.0 | 6.0 mg/L 盐水浸泡 |
| 高浓度组 | 8.0 | 8.0 mg/L 盐水浸泡 |

重要边界：

- `2.0 / 4.0 / 6.0 / 8.0 mg/L` 是盐水溶液浓度；
- 处理方式为清洗后的果实浸泡对应盐水浓度 3 分钟；
- 当前标签不是果实表面真实残留盐分；
- 后续模型只能表述为“预测浸泡浓度组别/浓度梯度响应”或“表面光学响应”，不能表述为“预测真实盐分含量”。

## 4. 命名规则

文件命名包含三类信息：

```text
[重复编号][浓度组]-[图像类型].png
```

### 4.1 重复编号

| 文件名前缀 | 含义 |
|---|---|
| 无前缀 | 第 0 次重复拍摄 |
| `-1` | 第 1 次重复拍摄 |
| `-2` | 第 2 次重复拍摄 |

示例：

- `盐浓度4.0mgL-0.png`
- `-1盐浓度4.0mgL-0.png`
- `-2盐浓度4.0mgL-0.png`

上述三张是 4.0 mg/L 组的三次重复拍摄中的原生图。

### 4.2 图像类型

| 后缀 | 含义 |
|---|---|
| `-0` | 原生图片 |
| `-1` | 聚焦图片 |

注意：

- `-0` 和 `-1` 来自同一实验条件下的同一光栅结果；
- 它们不是两个独立样本；
- 建模验证时不能把同一重复的 `-0` 和 `-1` 随机分到训练集和测试集两边，否则会产生数据泄漏。

## 5. 当前文件规模

当前目录共有 30 张 PNG 图像：

```text
5 个浓度组 x 3 个重复编号 x 2 种图像类型 = 30 张
```

独立重复数量应按每个浓度组 3 个重复计算，而不是按 6 张图片计算。

## 6. 数据风险

当前数据存在以下已知风险：

- 样本量很小，PLSR 极易过拟合；
- `-0` 与 `-1` 是同一样本双视图，不能当成独立样本；
- 盐水浓度单位为 mg/L，浓度较低，可能导致光学差异弱；
- 反射光受果皮曲率、湿润状态、反光、光源稳定性、拍摄角度、聚焦差异影响；
- 若光谱 ROI 不稳定，任何回归结果都不应被解释为有效检测能力。

## 7. 允许结论

在完成门禁分析前，只允许输出：

- 当前图像数据的组织方式；
- 图像质量、亮度、过曝、清晰度、ROI 可裁剪性；
- 光栅条纹是否可见；
- 同组重复是否稳定；
- 不同浓度组是否呈现初步可分趋势。

在完成严谨验证前，禁止输出：

- “可以准确测盐”；
- “可以识别水果表面盐分含量”；
- “PLSR 模型已经有效”；
- “该方法可用于食品安全检测”。
